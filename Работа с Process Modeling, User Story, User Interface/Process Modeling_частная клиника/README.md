**Кейс: Создание BPMN и UML State Machine диаграмм для приложения частной клиники**



**Легенда:** Сеть медицинских клиник хочет разработать приложение для пациентов. В приложении планируется реализовать функционал, который позволит пациентам регистрироваться в приложении клиники, самостоятельно находить необходимого врача, подходящее время для приёма и записываться на осмотр, минуя ручную работу администратора клиники.



**Задачи:**

1. На основе интервью с пациентами, врачами и администратором клиники создать [диаграмму бизнес-процесса AS IS](https://github.com/ylyubavskaya/portfolio/blob/main/%D0%A0%D0%B0%D0%B1%D0%BE%D1%82%D0%B0%20%D1%81%20Process%20Modeling%2C%20User%20Story%2C%20User%20Interface/Process%20Modeling_%D1%87%D0%B0%D1%81%D1%82%D0%BD%D0%B0%D1%8F%20%D0%BA%D0%BB%D0%B8%D0%BD%D0%B8%D0%BA%D0%B0/BPMN%20AS-IS.jpg) и [диаграмму состояний](https://github.com/ylyubavskaya/portfolio/blob/main/%D0%A0%D0%B0%D0%B1%D0%BE%D1%82%D0%B0%20%D1%81%20Process%20Modeling%2C%20User%20Story%2C%20User%20Interface/Process%20Modeling_%D1%87%D0%B0%D1%81%D1%82%D0%BD%D0%B0%D1%8F%20%D0%BA%D0%BB%D0%B8%D0%BD%D0%B8%D0%BA%D0%B0/UML%20State%20Machine%20Diagram.jpg).
2. Создать [диаграмму бизнес-процесса TO BE](https://github.com/ylyubavskaya/portfolio/blob/main/%D0%A0%D0%B0%D0%B1%D0%BE%D1%82%D0%B0%20%D1%81%20Process%20Modeling%2C%20User%20Story%2C%20User%20Interface/Process%20Modeling_%D1%87%D0%B0%D1%81%D1%82%D0%BD%D0%B0%D1%8F%20%D0%BA%D0%BB%D0%B8%D0%BD%D0%B8%D0%BA%D0%B0/BPMN%20TO-BE.jpg) с учётом автоматизации процесса записи на приём к врачу в приложении медицинской клиники.
